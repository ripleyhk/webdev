Blog Post - October 9th, 2020

This week, I learned all about Object-Oriented Programming in Javascript via
"The Flavors of Object-Oriented Programming (in JavaScript)", an article written by Zell Liew for CSS-Tricks.

Liew breaks Javascript Object-Oriented Programming down into 4 approaches:

  1. Using Constructor functions
  2. Using Classes
  3. Using Objects linking to other objects (OLOO)
  4. Using Factory functions

In order to help programmers make informed decisions on which approach to take, Liew also compares the approaches in the following areas:

  1. Inheritance
  2. Encapsulation
  3. this
  4. Event listeners

Terminology

Liew defines several useful terms:

  • Object-Oriented Programming — a way of writing code that allows you to create different objects from a common object
  • Instance — an object created from the common object, with its own properties
  • Inheritance — structuring code to have multiple levels of blueprints
  • Encapsulation — hiding certain pieces of information within the object so theyâ€™re not accessible

Constructor functions

Constructors are functions that contain a this keyword. They allow the programmer to store and access values for an instance. They can be called in JavaScript using the new keyword.

Here is a JavaScript constructor: 

function Human (firstName, lastName) {
  this.firstName = firstName
  this.lastName = lastName
}

And here is a new Human instance: 

const zell = new Human('Zell', 'Liew')
console.log(zell.firstName) // Zell
console.log(zell.lastName) // Liew

Classes

Liew claims that classes are an easier way to create constructors.

Here is what a Human class would look like in JavaScript: 

class Human {
  constructor(firstName, lastName) {
    this.firstName = firstName
    this.lastName = lastName
  }
}

The constructor may or may not be need to set firstName and lastName, depending on Human's inheritance. Objects can be instantialized the same with classes as they were with constructors.

Objects Linking to Other Objects

In OLOO, blueprints are defined like normal objects. Then, a method can be used to inialize the instance.

Here is a Human OLOO:

const Human = {
  init (firstName, lastName ) {
    this.firstName = firstName
    this.lastName = lastName
  }
}

The Object.create method is called to create the object. Then, the object's method is called to initalize its values.

This can be done like so:

const chris = Object.create(Human)
chris.init('Chris', 'Coyier')

console.log(chris.firstName) // Chris
console.log(chris.lastName) // Coyier

Factory Functions

Factory functions are functions that return an object.

Here is a Human factory function:

function Human (firstName, lastName) {
  return {
    firstName,
    lastName
  }
}

Factory functions do not require the new keyword.

Conclusion

Liew reached the following conclusions (these are arguably simplistic, but I'm not especially inclined to argue):

  1. Classes are better than Constructors
  2. OLOO is weird because of the Object.create part
  3. Classes and Factory Functions are easiest to use

Coming from a Java background, I'm inclined to agree with him. Classes in JavaScript work very similarly to classes in Java. OLOO has unnecessary complications. Factory functions are wonderful if you need to manage object instantialization. Overall, this article was very insightful.
