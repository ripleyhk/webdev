Blog Post - September 18th, 2020

I'll admit it: I love a good article with tips and tricks. I like throwing weird stuff together and seeing what won't frustrate users and clients away from my websites. That's why this week, I read "20 Essential CSS Tricks Every Designer Should Know" by James George from Web Designer Depot.

Did you know you can use the follow code to center text?

p { text-align: center; }

I was shocked by this revelation. How had I gone so long in my web developer career without such an innovative trick?

Of course, the article wasn't a complete waste of my time. Things started to heat up around Trick 13. :before. During my time with this trick, I learned how to use the :before CSS selector to preface an element with formatted content.

This can be implemented like so: 

warning:before { content: "Warning: "; color: red; }

And thus with just this HTML: 

<h2 class="warning">Don't believe everything you read on the internet.</h2>

We can achieve the following:

Don't believe everything you read on the internet.

It's incredibly helpful for creating uniform messages with limited code duplication. Just consider the alternative:

.warning { color: red;}

Sure. The CSS is simple enough. But now in HTML, we have to type:

<h2>
  <span class="warning2">Warning: </span>
  Don't believe everything you read on the internet.
</h2>

And of course. That's only for a single warning. Look how easy it is using :before: 

<h2 class="warning">Don't believe everything you read on the internet.</h2> 
<h2 class="warning">Vistors are <i>not</i> required to wear a mask to enter the store.</h2> 
<h2 class="warning">Vitamins are not FDA approved.</h2> 
<h2 class="warning">Don't blink.</h2>

To produce:

Don't believe everything you read on the internet.

Vistors are not required to wear a mask to enter the store

Vitamins are not FDA approved.

Don't blink.

As opposed to:

<h2>
  <span class="warning2">Warning: </span>
  Don't believe everything you read on the internet.
</h2>
<h2>
  <span class="warning2">Warning: </span>
  Vistors are <i>not</i> required to wear a mask to enter the store</h2>
</h2>
<h2>
  <span class="warning2">Warning: </span>
  Vitamins are not FDA approved.
</h2>
<h2>
  <span class="warning2">Warning: </span>
  Don't blink.
</h2>

Any opportunity for increased code elegance and decreased code duplication cuts down on the possible that the developer will make a mistake. Which means we can cut down the number of places we have to look to find the errors we will inevitably make while developing our products.
