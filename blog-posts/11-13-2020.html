Blog Post - November 13th, 2020

Admittedly, my web development skills still need some polishing, so this week, I read through Emmanuel Ohan's "Understanding Redux: The World's Easiest Guide to Beginning Redux" on FreeCodeCamp.org.

According to Ohan, Redux "helps you write applications that behave consistently." In short, Redux simplifies state management.

Ohan provides an incredibly informative ELI5, comparing Redux to withdrawing money from a bank, complete with vivid illustrations such as the one below:

The application state is held in the Redux store.

The only way to alter the state is by emitting an action, an object describing what happened. Below is an action to WITHDRAW $10,0000 from the bank: 

{ 
  type: "WITHDRAW_MONEY",
  amount: "$10,000"
}

Actions always have a type field describing the action. For example, the type above is WITHDRAW_MONEY.

The reducer communicates actions to the store via a process called dispatching. According to both Ohan and the official Redux documents, "Reducers are the most important concept in Redux."

Ohan then challenged me to create my magnum opus, a Hello World Redux app.

By the end, I had an App.js that looked like this:

import React, { Component } from "react";
import HelloWorld from "./HelloWorld";
import reducer from "./reducer";
import { createStore } from "redux";  

const initialState = { tech: "React " };
const store = createStore(reducer, initialState);

class App extends Component {
  render() {
    return <HelloWorld tech={store.getState().tech}/>
  }
  }

export default App;

And a reducer.js that looked like this:

export default (state) => {
   return state	
}

If there's anything I learned from this, it's that I still have plenty to learn. And of course, "Reducers are the most important concept in Redux."
